Nginx集群 负载均衡策略以及配置

轮询（Round Robin）根据Nginx配置文件中的顺序，依次把客户端的Web请求分发到不同的后端服务器

最少连接 请求被转发到连接数最少的服务器上

只是在upstream节点添加least_conn配置

IP地址哈希

只是在upstream节点添加ip_hash配置

权重

基于权重的负载均衡即Weighted Load Balancing

上面例子在服务器地址和端口后加weight=2配置，意味着每4个请求有2个分发到第一个服务器，第3个分发到第二个服务器，第3个请求分发到第三个服务器

ps:基于权重的负载均衡和基于IP地址哈希的负载均衡可以组合在一起使用。
